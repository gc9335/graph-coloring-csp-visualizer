# Graph Coloring CSP 答辩展板界面改造设计

## 目标

把当前网页从“功能完整的实验台”升级成“适合课程答辩和现场演示的学术展板式页面”。

本次改造的目标不是改变算法功能，而是在保留现有交互能力的前提下，让页面满足以下要求：

- 第一眼具有答辩展示感，而不是普通后台感。
- 信息主次清晰，老师能快速找到“主图、过程、剪枝”三个核心区域。
- 时间轴、伪代码、域状态矩阵、搜索树之间的联动关系更直观。
- 页面中文文案统一、无乱码，术语风格一致。
- 保持纯静态实现，继续可部署到 GitHub Pages。

## 范围

本次只改前端展示层，主要涉及：

- `index.html`
- `styles.css`
- `app.js`

本次不改动：

- 图着色算法逻辑
- 求解器数据结构
- GitHub Pages 部署方式
- 已有的搜索树缩放、拖拽、节点拖拽、时间轴回放等核心交互能力

## 设计方向

页面采用“学术展板风”的答辩展示方向：

- 基调：暖白纸张感背景、克制的深蓝灰主色、少量墨绿与朱砂橙作为强调色
- 观感：像高质量课程答辩展板，而不是 SaaS 后台或泛用仪表盘
- 重点：突出算法过程演示，尤其是“主图着色过程”“时间轴联动”“剪枝树”
- 视觉记忆点：中部主舞台和下方剪枝树形成双核心展示区

## 视觉语言

### 色彩

- 页面底色使用暖白与浅米色渐变，避免纯白发空。
- 主文字使用深蓝灰，保证学术感和可读性。
- 当前高亮使用深蓝色，表达“当前算法焦点”。
- 剪枝相关使用偏朱砂红/棕红色，表达“失败、剪枝、终止”。
- 成功路径使用偏墨绿，表达“找到解”。
- 候选色继续使用既定四色：
  - 朱砂红
  - 琥珀黄
  - 孔雀蓝
  - 松石青

### 字体

- 中文标题采用更有展示感的衬线风格。
- 正文保持高可读性的中文字体回退链。
- 英文眉题只作为辅助，不抢中文主标题。

### 质感

- 面板采用“轻玻璃感 + 纸张感”的折中风格。
- 阴影控制为柔和、低对比，不做夸张悬浮卡片。
- 线条和边框偏细，形成学术海报式秩序感。

## 页面结构

### 1. 顶部标题带

作用：承担“答辩封面”角色。

包含：

- 中文主标题：图着色 CSP 六版本动态实验台
- 英文小眉题：Constraint Search Studio
- 一句摘要：说明这是用于观察 V0-V5 图着色搜索过程的可视化页面
- 右侧关键状态摘要：当前算法、图规模、颜色上限、当前步骤等

设计要求：

- 中文主标题显著增大
- 摘要缩短并提升排版质量
- 当前状态信息用更整齐的统计块展示

### 2. 第一展示带：主舞台

左侧：

- 图着色主图，为整页视觉核心

右侧：

- 算法说明卡，包括：
  - 当前算法名称
  - 核心逻辑
  - 图形化展示说明
  - 时间复杂度
  - 空间复杂度
  - 当前事件
  - 调用栈
  - 当前启发式状态

设计要求：

- 主图面积明显大于说明区
- 主图背景干净，节点和边的对比更清晰
- 说明区分节更清楚，像讲解展板，不是普通长文本

### 3. 第二展示带：过程联动带

左侧：

- 固定高度、内部滚动的时间轴

右侧：

- 伪代码联动区

设计要求：

- 两者同一行，作为“过程解释”双栏
- 时间轴强调步骤顺序和事件类型
- 伪代码强调当前执行行和上下文关系

### 4. 第三展示带：约束传播带

上方：

- 域状态矩阵

下方：

- 搜索树 / 剪枝树

设计要求：

- 域矩阵作为“约束传播证据”
- 搜索树作为“剪枝过程亮点”
- 两者放在同一展示带中，体现“域变化导致搜索分支变化”

### 5. 底部控制区

保留当前控制功能，但压缩视觉权重，避免抢主体。

包含：

- 图选择 / 随机生成
- 颜色上限
- 速度
- 节点数 / 稠密度
- 播放与步进按钮
- 进度条

设计要求：

- 更统一的按钮形态
- 表单更精致
- 当前播放状态更清晰

## 关键组件改造

### 主图

- 节点改为更完整的实心着色展示
- 已确认颜色与试探状态必须视觉区分明确
- 当前节点、被拖拽节点、关联边分别有独立高亮层次
- 主图边线轻量化，避免与节点争夺注意力

### 时间轴

- 每个事件卡增加更明确的层次
- 当前事件卡使用强调底色和边框
- 不同类型事件可通过左侧色条或小标签区分
- 滚动条保留自定义样式，但要更精致

### 伪代码

- 采用“代码批注板”风格
- 当前高亮行更明确
- 非当前行对比降低，但仍然可读
- 行间距与缩进更稳定

### 域状态矩阵

- 行列关系更像实验表格
- 可用颜色块更整齐
- 当前受影响节点可有额外边框或轻高亮

### 搜索树 / 剪枝树

- 容器像展示窗，而不是普通滚动区
- 当前路径、成功路径、候选颜色分支、剪枝终止节点区分更清楚
- 找到目标解后，非解路径整体遮罩，正确路径高光保留
- 保留滚轮缩放与左键拖拽

## 中文与文案规范

需要统一修复和整理以下内容：

- 页面标题乱码
- 控件标签乱码
- 面板标题乱码
- 说明文案表述不统一

推荐统一术语：

- 主舞台
- 时间轴
- 伪代码联动
- 域状态矩阵
- 搜索树 / 剪枝树
- 当前事件
- 调用栈
- 启发式状态

按钮文案保持简洁：

- 随机生成图
- 重新计算
- 上一步
- 播放
- 下一步

## 动效策略

动效应克制，但必须有答辩展示感。

保留或增强的动效：

- 页面首屏轻微渐入
- 当前时间轴卡片切换时的状态过渡
- 伪代码高亮滑入
- 当前节点/路径高亮时的轻微强调
- 搜索树成功路径显现时的聚焦感

避免：

- 大面积炫光
- 高频闪烁
- 与教学信息无关的装饰动画

## 响应式要求

桌面端为主要目标，但页面需要兼容较窄屏幕。

要求：

- 宽屏下保持展板式双栏/分带布局
- 中等宽度下允许带状区域改为纵向堆叠
- 控制区在小宽度下可自动换行
- 搜索树和主图在窄屏下仍可操作，不出现内容不可达

## 可访问性要求

- 主体文字与背景保持足够对比度
- 所有按钮和输入应有清晰 hover/focus 状态
- 时间轴与树的滚动/拖动区视觉可感知
- 不依赖颜色单独传达状态，必要时补图标或边框差异

## 实现方案

### `index.html`

- 修复中文乱码
- 重组主结构为“标题带 + 主舞台 + 过程联动带 + 约束传播带 + 控制区”
- 为关键块增加更稳定的语义容器和标题层级

### `styles.css`

- 重建页面色彩和排版变量
- 提升标题区、面板、时间轴、树区、伪代码区的视觉质量
- 统一滚动区与交互区的样式
- 为成功路径、剪枝路径、当前路径建立更清晰的状态样式

### `app.js`

- 修正文案映射和中文标签
- 适配重组后的 DOM 结构
- 必要时调整滚动区同步、标题统计块、时间轴样式挂载逻辑

## 验收标准

改造完成后，应满足以下标准：

- 首页无中文乱码
- 第一眼能清楚看到“主图 + 时间轴/伪代码 + 搜索树”三大区
- 页面比当前版本更像答辩展示页，而不是普通实验工具页
- 时间轴和伪代码联动更易读
- 搜索树更像教学重点展示区
- 所有现有交互功能保持可用
- 仍然可以作为纯静态网页部署到 GitHub Pages

## 风险与控制

### 风险 1：视觉增强过度，压过内容

控制：

- 所有强调只服务于“当前状态、剪枝、成功路径”
- 不增加与教学信息无关的大装饰

### 风险 2：重排布局后联动失效

控制：

- 尽量保留现有 DOM id
- 优先做结构包装，不随意重命名关键挂载点

### 风险 3：样式提升后页面变重

控制：

- 优先 CSS 方案
- 不引入额外框架
- 不增加构建步骤
